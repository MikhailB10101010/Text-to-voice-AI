# Автоматизированный конвейер машинного обучения (Lab 1)

Этот проект представляет собой простейший MLOps-конвейер, который автоматизирует процесс создания данных, их предобработки, обучения модели и её тестирования. Весь процесс управляется одним bash-скриптом.

## Описание

Конвейер решает задачу бинарной классификации на синтетических данных.
Процесс состоит из следующих этапов:
1.  **Генерация данных**: Создание синтетического набора данных с добавлением шума и аномалий. Разделение на обучающую (`train`) и тестовую (`test`) выборки.
2.  **Предобработка**: Масштабирование признаков (StandardScaler) на основе обучающей выборки.
3.  **Обучение**: Обучение модели логистической регрессии и сохранение её в формате `.pkl`.
4.  **Тестирование**: Загрузка обученной модели, предсказание на тестовых данных и вывод метрики Accuracy.

## Структура проекта

Файлы должны находиться в корневой директории `lab1/`:

*   **`data_creation.py`**: Скрипт генерации синтетических данных. Создает папки `train` и `test` и сохраняет туда `.csv` файлы.
*   **`data_preprocessing.py`**: Скрипт предобработки. Загружает «сырые» данные, стандартизирует их и сохраняет новые файлы `*_preprocessed.csv`.
*   **`model_preparation.py`**: Скрипт обучения. Обучает `LogisticRegression` и сохраняет модель в файл `model.pkl`.
*   **`model_testing.py`**: Скрипт валидации. Проверяет модель на тестовой выборке и выводит итоговую точность в консоль.
*   **`pipeline.sh`**: Bash-скрипт, который устанавливает необходимые библиотеки Python и последовательно запускает все этапы пайплайна.

## Требования

*   ОС: Linux (Ubuntu/Debian) или macOS.
*   Установленный Python 3.
*   Установленный менеджер пакетов `pip`.

## Установка и запуск

1.  Убедитесь, что вы находитесь в директории `lab1`:
    ```bash
    cd lab1
    ```

2.  Сделайте скрипт запуска исполняемым:
    ```bash
    chmod +x pipeline.sh
    ```

3.  Запустите конвейер:
    ```bash
    ./pipeline.sh
    ```

> **Примечание:** Скрипт `pipeline.sh` автоматически попытается установить необходимые библиотеки (`pandas`, `scikit-learn`, `numpy`) через `pip`. Рекомендуется запускать проект в виртуальном окружении, если вы не хотите устанавливать пакеты глобально.
