# Лабораторная работа №4: Версионирование данных с DVC

## Цель работы
Освоить базовые принципы версионирования датасетов с помощью утилиты DVC, создать несколько версий данных и настроить удаленное хранилище.

## Структура данных
В рамках работы создано 3 версии датасета Titanic:
1. data_v1.csv – исходные "сырые" данные (891 строка, есть пропуски в признаке `Age`).
2. data_v2.csv – данные после предобработки: пропуски в `Age` заполнены медианой в разрезе `Pclass`.
3. data_v3.csv – данные после кодирования: категориальный признак `Sex` преобразован с помощью One-Hot Encoding.

## Настройка удаленного хранилища (Remote)
Из-за сетевых ограничений при попытке настройки облачных хранилищ (Google Drive / Yandex Disk WebDAV) было принято решение использовать локальное хранилище на диске `D:`. Это полностью сохраняет функциональность DVC и демонстрирует понимание принципов работы утилиты.

Настройка выполнялась следующими командами:
```bash
dvc remote add -d myremote D:/lab4_dvc_cache
